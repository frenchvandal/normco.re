## Emoji in Context

Emojis work naturally in regular text flow:

> :bulb: **Pro tip**: Use emojis sparingly in technical documentation to
> maintain professionalism while adding visual interest.

### Status Indicators

Emojis are great for status indicators in documentation:

- :white_check_mark: Feature implemented
- :construction: Work in progress
- :x: Not supported
- :warning: Deprecated

### Commit Message Style

Many projects use emojis in commit messages following conventions like
[Gitmoji](https://gitmoji.dev/):

| Emoji                  | Type                 | Example                                        |
| ---------------------- | -------------------- | ---------------------------------------------- |
| :sparkles:             | New feature          | :sparkles: Add dark mode toggle                |
| :bug:                  | Bug fix              | :bug: Fix navigation overflow                  |
| :memo:                 | Documentation        | :memo: Update README                           |
| :art:                  | Style/format         | :art: Format code with Prettier                |
| :rocket:               | Deploy               | :rocket: Deploy version 2.0.0                  |
| :recycle:              | Refactor             | :recycle: Simplify auth logic                  |
| :lock:                 | Security             | :lock: Fix XSS vulnerability                   |
| :globe_with_meridians: | Internationalization | :globe_with_meridians: Add French translations |

## Emoji Accessibility

When using emojis, consider accessibility:

1. **Don't rely solely on emojis** to convey meaning
2. **Use descriptive text** alongside emojis
3. **Avoid emoji overload** — too many can be distracting
4. **Test with screen readers** — some may read emoji names aloud

## Summary

Emojis are a fun and expressive way to enhance your content. Use them
thoughtfully to add personality without compromising clarity or accessibility.
